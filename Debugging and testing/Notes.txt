Errors:- An error is a mistake in a program that prevents it from compiling , running
or producing correct output 

    1) Syntax error:- This occurs when rules of programming language when rules of programming
    language breaks and the compiler dectects it 
        ex:- printf("Hello...") --> Missing semicolon
    
    2) Logical errors:- The program runs but give wrong output 
        ex:- int a=5 , b=0;
             printf("%d",a/b); --> Division is not possible 
    
    3) Run Time error:- Error that occurs that occurs while running the program 
        ex:- int a=5 , b=0;
             printf("%d",a/b); --> Division is not possible

    4) Symantic error:- Meaning of the statement is incorrect even though the 
    syntax is correct 
        ex:- int a = "Hello"; --> Datatype mismatch 
    
    5) Linker error:- Occurs during linking state happens when function defination is 
    missing 
        ex:- calling of function 

    6) Compilation error:- Genrally error shown by compiler

Debugging Techniques:- Debugging is the process of finding and fixing errors or bugs in a program to make 
it run correctly

    1) Code review:- Carefully read your code line by line to find syntax error 
    and Logical mistakes 

    2) Using print stmt:- In printf to check the value of diffrent varibles 

    3) Dry run or tracing:- Manually flow the code logic with sample input helps to 
    dectect Logical error before running the program 

    4) Using debugger:- Tools like GDB in C and C++ the debugger allows setting break 
    points, step by step execution inspecting varible value at runtime 

    Rubber duck debugging :- It explain the code line by line to someone else or even 
    a rubber duck 

    Unit testing:- The small modules are checked independently to check correctness 
    the large programs are divided into small modules known as units for testing 
--------------------------------------------------------------------------------------
Types of testing:-
    White box testing---->
        1) Unit Testing:- Testing indivisual units or modules of a program (When everything 
        is diffrent)
        
        2) Integration Testing:- Testing the combinations of two or more modules togather 
        It checks integration between modules (OOP) 
        
        3) System Testing:- Testing enitre system as a whole 
        --> Ensures that everything works from start to end 
        
        4) Acceptance Testing:- Performed by cliet and end user
        --> Checks if system meets user requirements
        
        5) Functional Testing:- Test the Functionality of software based on what the
        system is supposed to do.
        
        6) Non Functional Testing:- Test the Performace , speed , security , usablity execution
        
        7) White Box Testing:- Tester knows the internal code structure(Logic) 
    
    Black Box Testing(Only outer testing):- Tester does not know the internal code structure 
    --> Test only input and output.
